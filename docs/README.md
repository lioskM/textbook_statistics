# 統計学教科書 レビューパイプライン

## 概要

教科書本文のドラフトに対し、4体のエージェントが直列にレビューを行う。
各エージェントは指摘と修正案を出力し、次のエージェントはその修正済みテキストを入力として受け取る。

## パイプライン

```
原稿（LaTeX）
  │
  ▼
① 統計的正確性（agents/01_statistical_accuracy.md）
  │  定義・数式・前提条件の誤り、用語の表記ゆれを検出・修正
  ▼
② 構造・整合性（agents/02_structural_consistency.md）
  │  概念導入順序、章構成案との整合、6ステップ充足を検出・修正
  ▼
③ 文章品質（agents/03_writing_quality.md）
  │  アンチパターン、filler、hedge、メタ宣言を検出・修正
  ▼
④ リズム・グルーヴ（agents/04_rhythm.md）
  │  文の長短バランス、読者との対話感、冗長箇所の削減
  ▼
最終稿 → Ryosukeが確認
```

## 順序の根拠

- ①が最初：統計的に間違っている文を磨いても無駄
- ②が次：構造的に不要な段落の削除や順序入れ替えが後段の文体修正を無駄にするのを防ぐ
- ③が④の前：fillerや冗長表現を削ってからでないとリズムの判断ができない
- ④が最終段：リズムは前段までの修正を踏まえた上でしか判断できない

## 各エージェントの参照ファイル

| エージェント | 必須参照 | 任意参照 |
|---|---|---|
| ① 統計的正確性 | 要件定義書 §4.4, §4.8 / latex_reference.md | 対象章のLaTeX |
| ② 構造・整合性 | 要件定義書 §5.2, §6 / 章構成案 v3 / latex_reference.md | 前章までのLaTeX |
| ③ 文章品質 | 要件定義書 §4.4, §4.5, §4.6 | — |
| ④ リズム | リズム原則 v1（R1〜R10 + Before/After） | — |

※ 節番号は要件定義書v3.0の体系に準拠。

## ディレクトリ構成

```
textbook-agents/
├── README.md
├── review.sh                  # 実行スクリプト
├── agents/
│   ├── 01_statistical_accuracy.md
│   ├── 02_structural_consistency.md
│   ├── 03_writing_quality.md
│   └── 04_rhythm.md
├── latex_reference.md          # LaTeX環境名・章ファイル対応表
├── 要件定義書_v3_0.txt         # ※ 配置が必要
├── 統計学教科書_章構成案_v3.md  # ※ 配置が必要
└── リズム原則_v1.md            # ※ 配置が必要
```

## 実行方法（Claude Code想定）

```bash
# セットアップ
chmod +x review.sh

# 参照ファイルをこのディレクトリに配置
# - 要件定義書_v3_0.txt
# - 統計学教科書_章構成案_v3.md
# - リズム原則_v1.md

# 実行
./review.sh data_summarization.tex
```

**注意**：`claude --print -s -a` のオプション体系は `claude --help` で確認すること。上記は想定される使い方であり、実際のCLIインターフェースに合わせて調整が必要。

## 出力フォーマット（全エージェント共通）

各エージェントは以下の形式で出力する：

```
## 変更一覧

1. [該当箇所] 変更内容の説明
2. ...

## 修正済みテキスト

（修正を反映したLaTeX全文）
```

変更がない場合は「変更なし」と明記し、入力テキストをそのまま出力する。
