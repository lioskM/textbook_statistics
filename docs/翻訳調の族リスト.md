# 翻訳調の族リスト v7

配置: `docs/翻訳調の族リスト.md`（v6 を置き換え）

## 位置づけ

- 本書の日本語から翻訳調を中心とする不自然な文を検出するための文書。**検出専用**であり、生成時にこのリストを遵守させる用途には使わない（機序レベルの生成指示は空転することが確認済み）。
- 運用サイクル: 生成 → 族との照合（判別タスク）→ 修正。機械走査で候補を挙げ、判定と書き直しは Chat＋Ryosuke が行う。書き直し案の生成は Codex に委ねないが、判定済み採取の再整理や走査実装の知見に基づく**リスト設計の提案**は受け付ける（採否の判定は Chat＋Ryosuke。v7 の族5・非該当条件・実装規定は Codex 提案を基に採用）。
- **判定の手順則**:
  1. 慣用句・定着表現の**存在**を確認しただけで正用と判定しない。当該の**表面形そのもの**が、支え（副詞・係り結び・漢字）を保ったままそのレジスタで流通しているかまで確認する。慣用の残骸は慣用ではない（第7章「ここでは置く」の教訓）。
  2. **口語表現それ自体は欠陥ではない**。検出対象は日本語として不自然なもの（翻訳調・結合破綻・構文破綻）であり、話し言葉調の語彙（「みたいだ」等）は周囲の文体との釣り合いで個別判断する。口語と不自然の線引きを網羅的に規則化しない。
- リズム原則v4とは独立の文書。v4 §3「翻訳調を避ける」の名指し例のうち、本文で再発が採取されたものから順に族へ吸収する（体言止めの羅列は族4として吸収済み。能力獲得構文・手段明示の「によって」は本書「非該当条件集」に判定基準を吸収済みで、走査対象として維持）。
- 族とは、発生機序で括った癖のグループ。各族は次の4点で構成する。
  1. 機序の定義
  2. 日本語での正しい書き方
  3. 実例（**例示であり網羅ではない**。本文から採取した「採取実例」と、機序から作った「想定例」を区別して記す）
  4. **機序に該当するものは、表面形を問わずすべて禁止**（リストにない表面形であることは非該当の理由にならない。該当判定は Chat＋Ryosuke が行う）

修正の判断基準は `docs/修正方針.md` §1 の優先順位に従う。族への機械的該当だけを理由に、内容や余韻を壊す書き換えをしない。

## 族1: 教科書の書き言葉に流通していない運動・動作の比喩で論理を書く

**機序**: 英語は計算・手続き・論証・共変関係を物理的な運動（halt, run, move together, break, map）で語ることが多く、その運動表現を直写すると、日本語の数理の書き言葉に存在しない言い回しが混入する。

**該当条件**: 欠陥は比喩であること自体ではなく、**その運動表現が日本語の数理・統計の書き言葉レジスタで流通していないこと**にある。判定は次の順で行う。

- 「議論を進める」「式を展開する」「解を導く」「分布に従う」、変数操作の「動く文字」「$b$だけが動く」、自由度の「自由に動ける」のように書き言葉でも定着した表現は非該当。
- 日常の話し言葉としては自然でも、書き言葉レジスタでは別の語彙が標準の場合は該当（共変の裸の「一緒に動く」→「連動する」「増減の向きが揃う」等）。
- **「動く」の3系統の切り分け**:
  1. 単位換算・定義変更で値が「動く」→ **該当**（何も変動しておらず再計算されただけ。「変わる」で書く）
  2. 共変の裸の「一緒に動く」→ **該当**（「連動する」等で書く）
  3. 繰り返し・取り直し・時間に伴う変動（「取り直すたびに平均が動く」「動くのは区間のほう」「μは動かない」）→ **非該当**（「相場が動く」「数字が動く」として定着した変動のメタファー。空間的な位置の変動を図が裏づける場合はなお強い）
- **装置の例外**: カギ括弧つきで日常語として提示し定式化する装置（第8章「一緒に動く」）の初出と呼び返しは非該当。裸で運用語彙に流用した箇所のみ該当。
- 判定に迷う場合は web 検索で確認する。その際、手順則1に従い、慣用の存在ではなく**当該表面形の流通**を確認する。

**日本語での正しい書き方**: 何が起きるかを、対象そのものの振る舞いとして直接述べる。式なら「値を返す」「実行できる」「成り立たない」、共変なら「連動する」「増減の向きが揃う」など。慣用の表現があるなら**完全形で**使う（「待ったをかける」「ひとまず脇に置く」等）。

**採取実例**:
- 「計算は止まらず」→「公式に入れれば傾きと切片の数字が出てくる」（halt の比喩。第9章$R^2$節）
- 「計算式自体は説明変数を増やせば動く」→「計算自体は, 説明変数を増やしても同じ手順で実行できる」（run の比喩。第10章線形性）
- 「この結論を止めた」→「この結論に待ったをかけた」（halt を論証対象に適用。第10章生存者バイアス）
- 「歪みを地図にした」→「歪みの見取り図を描いた」（map out の動詞用法の直写。第10章終節）
- 「同じ方向に動く傾向が強いのか」→「増減の向きが揃う傾向が強いのか」（move together の共変用法。第8章）
- 「値そのものがこれだけ動くと」→「これだけ変わってしまうと」（単位換算の系統1。第8章）
- 「どれくらい一緒に動くかは表しても」→「どれくらい連動するかは表しても」（裸の共変用法。第9章冒頭）
- 「どれくらい一緒に動くかを表す相関係数」→「どれくらい連動するかを表す相関係数」（同上。第2章の先出し紹介）
- 「二つの量が一緒に動いていることは見えても」「視聴回数と一緒に動いている量」→「連動していること」「連動している量」（同上。第3章散布図・箱ひげ図）
- 「片方を止めて」「$b$は止めてある」→「固定して」「固定してある」（同一操作に「固定」と「止める」が混在。標準語彙の「固定」に統一。第9章導出）
- 「手順が壊れていれば」→「手順の前提が崩れていれば」（break を手順という準論理的対象に適用。第5章）

**想定例**:
- 「証明が壊れる」「議論が走る」のような、計算・論証を主語にした未流通の運動動詞

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族2: 参照の完全明示

**機序**: 英語は既出対象を再参照するとき位置情報つきで明示する（the equation above, as shown earlier）。これを直写すると、読者の頭で参照が生きている場面でも「先に示した」「上の」「同じ」を付けてしまう。日本語では参照が生きている限り指示詞か本文で与えた呼び名で軽く受けるのが自然で、位置の明示は参照が切れかけた場面に限られる。

**該当条件**: 直前ないし近傍の対象で、読者の頭に参照が生きているのに位置情報つきで受けている場合が該当。間に別の話題が挟まり参照が切れかけている場面での「先に示した」「前章で見た」「先ほどの$t$検定」「先ほどの図」（節をまたぐ再参照）は非該当。

**日本語での正しい書き方**: 参照が生きている対象は「この式」「その値」の指示詞で受けるか、本文で与えた呼び名（「分解の式」等）で受ける。小見出しをまたぐ場合は呼び名のほうが強い。

**採取実例**:
- 「先に示した回帰式に」→「この式に」／「同じ式に代入すると」→「この式に代入すると」（直前の式を受ける場面）
- 「上の式の各項の期待値を求めて」→「分解の式の各項の期待値を求めて」（小見出し越しは呼び名で受ける。第7章自由度）
- 「その分母として上の式が現れている」→「その分母としてこの式が現れている」（「この式は〜」の鎖が生きている場面。第7章t検定）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族3: 従属節を読点で重ねて流し込む構文

**機序**: 英語の分詞構文・節連結（-ing, which, while）をそのまま写すと、従属節を読点で連ねて一文に詰める構造になる。日本語の説明文はこの負荷に耐えず、特に数式へ流れ込む文で顕著に不自然になる。

**該当条件**: 読点で連ねられた節の各々が独立の事態を述べており、日本語なら文を切って書く内容である場合が該当。並列の列挙、条件・理由の複文、対句、引用の区切りといった日本語本来の読点機能は非該当。数式モード内のカンマは読点ではない。

**日本語での正しい書き方**: 文を切る。一文一事に分解し、接続は文頭の接続詞か指示詞で受け直す。

**採取実例**:
- 「〜を入れても, 止まらず,」（族1との複合例。「〜を入れてみる. それでも式は値を返す.」のように切る）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族4: 受けのない体言止めの羅列

**機序**: 英語の名詞句断片文（fragment: "Multiple testing. Bootstrapping. Regression."）を直写すると、名詞列だけで終わる文が生じる。日本語の説明文では、名詞列は述語で閉じるか、直後の文が指示詞等で受けて初めて座りが決まる。

**該当条件**: 地の文で、名詞列だけの文を受ける述語も直後の受けの文もなく、列が宙に浮いている場合が該当。次は非該当:

- 直後の文が「どれも〜」「いろいろな要因が〜」「この見方から〜」のように列・断片を受けている提示→受けの構造（命名の直前に間を作る断片も含む）
- 「〜こと.」を連ねて直後の文で受ける列挙
- 図表キャプション・書誌情報（脚注の文献表記）・見出し（日本語の慣用として体言止めが標準）
- **助動詞「だ」を省略した名詞述語文**（述語がある）: 「偏差は正, 小さければ負.」「ひとつは〜こと.」「返送されたのは約240万通.」「成功回数なら二項分布.」

**日本語での正しい書き方**: 述語を付けて閉じる（「〜といった話題だ」）か、直後に受けの文を置く。

**採取実例**:
- 「多重比較の制御, ブートストラップ, 重回帰, 時系列モデルや階層モデル.」→「〜といった話題だ.」（次文が別話題に移り、列を受ける述語がなかった。第10章終節）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族5: 一文に積んだ節・引用・モダリティで係り受けが崩れる

**機序**: 一つの文に節構造・引用・可能表現・推量などを積んだ結果、助詞の係り先や文の中心（何がどうする）が最後まで残らず、係り受けが崩れる。**注記**: 本族は現象側（係りの崩壊）で定義する。英語構文の直写が原因の場合も、単なる推敲不足の場合もあり、生成原因を問わず該当とする。したがって本族は翻訳調に限らない欠陥類型である。

**該当条件（確認点）**: 次のいずれかに答えられない文が該当。

1. 「は／が／を」の受け先が文末まで残っているか
2. 文の中心を「何がどうする」に戻せるか
3. 引用の「という」が本当に引用を受けているか
4. 可能・推量・受身を一文に重ねすぎていないか

**日本語での正しい書き方**: 文の中心を一つに定め、積み過ぎた要素は文を分けるか削る。

**採取実例**（いずれも第3章冒頭。同一区画に集中して発生した）:
- 「問題はデータの可視化を, そのデータの全体的な情報として意味を持つようにしなければならないということのようだ」→「問われているのは, データの全体的な情報として意味を持つ可視化をどう選ぶか, ということのようだ」（「を」の係り先が消失）
- 「全体の情報も見ることができるとデータの傾向を確認できたりしそうだが」→「全体の情報まで見えればデータの傾向をつかむのに役立ちそうだが」（可能表現の重複と「たり」の単用）
- 「〜示してみよう, というと」→「〜示すなら」（自分の意図の引用を「というと」で受ける接続の破綻）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 非該当条件集（頻出パターンの判定基準）

5回の走査で繰り返し判定した基準の成文化。候補に挙がっても、以下は非該当とする。

- **`によって`**: 「人によって違う」「ジャンルによって違う」「処理によって変わる」のような比較軸・依存関係・原因・根拠（「中心極限定理によって」）を担う用法は正用。該当するのは v4 §3 の名指しどおり、**削っても意味の変わらない手段明示**のみ。第1回パスでは候補十数件が全件正用だった。
- **`見えてくる`・`手に入`**: 可視化・統計量の組み合わせなどで実際に理解や情報が段階的に得られる文脈は正用（「分布の形として見えてくる」「データを手に入れる」）。該当するのは、手法の効能を後付けで宣言するだけの**能力獲得構文**のみ。
- **`1つ`**: 「1つの数」「1つ選ぶ」など可算対象は正用。該当するのは金額・長さ・連続量の**増分**に助数詞として使う場合のみ（「製作費が1つ増える」）。
- **文頭の `それは`・`これは`**: 問い→答え（「なぜか. それは〜からだ」）、直前対象の指示・否定（「これは違う」「これは見立てではなく」）は正用。該当するのは前文を受け直して同じ内容を言い直す形のみ。
- **`置く`**: 仮定・条件を置く、基準を置く、文書のある場所に置く（「証明はコラムに置く」）、完全形の慣用（「ひとまず脇に置く」）はすべて正用。

## 族未満の採取（機序の括り待ち）

族に括るには実例が足りないが、修正が確定した採取。同種が再発したら族に昇格させる。

- **結合の立たない動詞・目的語（2件・次の1件で族昇格）**: 「比較が言い切れる」→「問いにはっきり答えられる」（第8章）／「同じ構造をたどる」→「同じ道筋をたどる」（第6章）。文法的には解析できても、日本語の動詞がその主語・目的語を取らない。族1の運動比喩と異なり比喩でなくても起こるため、走査語を増やすのではなく通読時の確認項目（下記「通読の手がかり」1）で拾う。
- **慣用句の残骸**: 「ここでは置く.」→「ここでは立ち入らない.」（第7章Welch節）
- **話し言葉レジスタの混入**: 「うまくいかないみたいだ」→「うまくいかないようだ」（口語自体は欠陥ではなく、本書の地の文の推量が「ようだ」で統一されている釣り合いに基づく個別修正。第3章冒頭）
- **place/placement の直写（2件・昇格候補）**: 「予測値に対して置く図」→「プロットした図」（第9章）／「という配置」→削除（第8章）
- **one を「1つ」と直写（2件・昇格候補）**: 「1つ動かした」「1つ増えるごとに」→「1単位」（第9章）
- **同格コンマの直写**: 「点$(\bar{x}, \bar{y})$, データの重心にあたる点を」→「データの重心にあたる点$(\bar{x}, \bar{y})$を」（第9章）
- **読者の身体反応の予告**: 「意外なところで手が止まる」→「線の置き場所が意外に決まらない」（第9章）
- **語法の無理**: 「1単位散らばるごとに」→「散らばり1単位あたり」（第9章）
- **逆の意味に取れる多義**: 「誰も因果を疑わないが」→「因果を本気にする人はいないが」（第8章）
- 「その約束の場所がここになる.」→「その約束をここで果たす.」（第10章）
- 「ここで本書はいったん筆を置く.」→「ここでいったん筆を置く.」（第10章）
- 「モデルを当て,」→「モデルを当てはめ,」（第10章）

## 通読の手がかり

通読時の最低限の照合項目。**網羅ではない**。通読の主務はこの項目の点検ではなく、リスト外の違和感を自分の判断で挙げることにある（第1回パスの実測: 機械走査の真陽性9件に対し通読23件。その多くは事前のリストにない違和感から出た）。項目照合に検出を狭めないこと。

1. 主語と動詞、動詞と目的語の結合が日本語として立っているか
2. 助詞の係り先が文末まで残っているか
3. 引用・可能・推量を外しても文の中心が残るか
4. 英語なら自然な比喩や位置参照を、そのまま移していないか

## 走査の手がかり（機械走査用・非網羅）

grep 候補。ヒットは判定前の候補にすぎず、非ヒットは非該当を意味しない。走査時は本リストに加え、リズム原則v4 §3 の未吸収の名指し例も照合対象とする。

**実装規定**:
- 図表キャプション（`\caption`）内と脚注内の書誌情報は族4の対象外。
- 数式モード（`$...$`, `\[...\]`, equation 環境）内のカンマは読点として数えない。
- TeX の空白命令 `\,` は読点に数えない。
- 句点のない `\item` どうしを一文として連結しない。
- 表示数式末尾の `.` は文の終端として扱う。

**パターン**:
- 族1: `止ま`, `止め`, `動く`, `動い`, `動き出`, `走`, `壊れ`, `地図に`（計算・式・議論・結論・変数の共変が絡む文脈のみ。カギ括弧内の装置引用と繰り返し変動の系統3は除外してよいが、判別が面倒なら全件拾う）
- 族2: `先に示した`, `先ほど`, `上で見た`, `上の式`, `前に述べた`, `同じ式`, `同じ値`
- 族3: 一文中の読点3個以上かつ数式接続あり（偽陽性率が高いため優先度低）
- 族4: 句点で終わり述語を含まない地の文
- 族5: 機械パターン化は困難。通読の手がかり1〜3で拾う
- 族未満: `置く`, `配置`, `1つ`（連続量・金額の文脈のみ）, `たどる`, `言い切れ`
