# 翻訳調の族リスト v3

配置: `docs/翻訳調の族リスト.md`（v2 を置き換え）

## 位置づけ

- 本書の日本語から翻訳調を検出するための文書。**検出専用**であり、生成時にこのリストを遵守させる用途には使わない（機序レベルの生成指示は空転することが確認済み）。
- 運用サイクル: 生成 → 族との照合（判別タスク）→ 修正。機械走査で候補を挙げ、判定と書き直しは Chat＋Ryosuke が行う。書き直し案の生成を Codex に委ねない（生成経路が同じで盲点を共有するため）。
- リズム原則v4とは独立の文書。v4 §3「翻訳調を避ける」の名指し例のうち、本文で再発が採取されたものから順に族へ吸収する（体言止めの羅列は族4として吸収済み）。未吸収の名指し例（能力獲得構文、「それは／これは」の受け直し、削っても意味の変わらない手段明示、擬人化など）は走査の照合対象として引き続き有効。
- 族とは、発生機序で括った癖のグループ。各族は次の4点で構成する。
  1. 機序の定義
  2. 日本語での正しい書き方
  3. 実例（**例示であり網羅ではない**。本文から採取した「採取実例」と、機序から作った「想定例」を区別して記す）
  4. **機序に該当するものは、表面形を問わずすべて禁止**（リストにない表面形であることは非該当の理由にならない。該当判定は Chat＋Ryosuke が行う）

修正の判断基準は `docs/修正方針.md` §1 の優先順位に従う。族への機械的該当だけを理由に、内容や余韻を壊す書き換えをしない。

## 族1: 教科書の書き言葉に流通していない運動・動作の比喩で論理を書く

**機序**: 英語は計算・手続き・論証・共変関係を物理的な運動（halt, run, move together, map）で語ることが多く、その運動表現を直写すると、日本語の数理の書き言葉に存在しない言い回しが混入する。

**該当条件**: 欠陥は比喩であること自体ではなく、**その運動表現が日本語の数理・統計の書き言葉レジスタで流通していないこと**にある。判定は2段階になる。

- 「議論を進める」「式を展開する」「解を導く」「分布に従う」、変数操作の「動く文字」「$b$だけが動く」のように書き言葉でも定着した表現は非該当。
- 日常の話し言葉としては自然でも、書き言葉レジスタでは別の語彙が標準の場合は該当（第8・9章の共変用法: 裸の「2つの量が一緒に動く」「同じ方向に動く」「値が動く」→「連動する」「増減の向きが揃う」「値が変わる」）。
- **装置の例外**: カギ括弧つきで日常語として提示し定式化する装置（第8章「一緒に動く」）の初出と呼び返しは非該当。裸で運用語彙に流用した箇所のみ該当。
- 判定に迷う場合は、その言い回しが日本語の数理・統計の文章で実際に使われているかを web 検索で確認する。

**日本語での正しい書き方**: 何が起きるかを、対象そのものの振る舞いとして直接述べる。式なら「値を返す」「実行できる」「成り立たない」、共変なら「連動する」「増減の向きが揃う」など。慣用の表現があるならそれを使う（「待ったをかける」等）。

**採取実例**:
- 「計算は止まらず」→「公式に入れれば傾きと切片の数字が出てくる」（halt の比喩。第9章$R^2$節）
- 「計算式自体は説明変数を増やせば動く」→「計算自体は, 説明変数を増やしても同じ手順で実行できる」（run の比喩。第10章線形性）
- 「この結論を止めた」→「この結論に待ったをかけた」（halt を論証対象に適用。第10章生存者バイアス）
- 「歪みを地図にした」→「歪みの見取り図を描いた」（map out の動詞用法の直写。第10章終節）
- 「同じ方向に動く傾向が強いのか」→「増減の向きが揃う傾向が強いのか」（move together の共変用法。第8章）
- 「値そのものがこれだけ動くと」→「これだけ変わってしまうと」（fluctuate の直写。第8章）
- 「どれくらい一緒に動くかは表しても」→「どれくらい連動するかは表しても」（裸の共変用法。第9章冒頭）
- 「片方を止めて」「$b$は止めてある」→「固定して」「固定してある」（同一操作に「固定」と「止める」が混在。標準語彙の「固定」に統一。第9章導出）

**想定例**:
- 「証明が壊れる」「議論が走る」のような、計算・論証を主語にした未流通の運動動詞

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族2: 参照の完全明示

**機序**: 英語は既出対象を再参照するとき位置情報つきで明示する（the equation above, as shown earlier）。これを直写すると、読者の頭で参照が生きている場面でも「先に示した」「同じ」を付けてしまう。日本語では参照が生きている限り指示詞で軽く受けるのが自然で、位置の明示は参照が切れかけた場面に限られる。

**該当条件**: 直前ないし近傍の対象で、読者の頭に参照が生きているのに位置情報つきで受けている場合が該当。間に別の話題が挟まり参照が切れかけている場面での「先に示した」「前章で見た」は非該当（それが日本語でも正しい用法）。

**日本語での正しい書き方**: 参照が生きている対象は「この式」「その値」で受ける。

**採取実例**:
- 「先に示した回帰式に」→「この式に」
- 「同じ式に代入すると」→「この式に代入すると」（いずれも直前の式を受ける場面で採取）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族3: 従属節を読点で重ねて流し込む構文

**機序**: 英語の分詞構文・節連結（-ing, which, while）をそのまま写すと、従属節を読点で連ねて一文に詰める構造になる。日本語の説明文はこの負荷に耐えず、特に数式へ流れ込む文で顕著に不自然になる。

**該当条件**: 読点で連ねられた節の各々が独立の事態を述べており、日本語なら文を切って書く内容である場合が該当。並列の列挙、条件・理由の複文、対句、引用の区切りといった日本語本来の読点機能は非該当。数式モード内のカンマ（座標対・数式の区切り）は読点ではない。

**日本語での正しい書き方**: 文を切る。一文一事に分解し、接続は文頭の接続詞か指示詞で受け直す。

**採取実例**:
- 「〜を入れても, 止まらず,」（族1との複合例。「〜を入れてみる. それでも式は値を返す.」のように切る）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族4: 受けのない体言止めの羅列

**機序**: 英語の名詞句断片文（fragment: "Multiple testing. Bootstrapping. Regression."）を直写すると、名詞列だけで終わる文が生じる。日本語の説明文では、名詞列は述語で閉じるか、直後の文が指示詞等で受けて初めて座りが決まる。

**該当条件**: 地の文で、名詞列だけの文を受ける述語も直後の受けの文もなく、列が宙に浮いている場合が該当。次は非該当:

- 直後の文が「どれも〜」「集団内の点は〜」のように列を受けている提示→受けの構造
- 「〜こと.」を連ねて直後の文で受ける列挙
- 図表キャプション・書誌情報（脚注の文献表記）・見出し（日本語の慣用として体言止めが標準）
- 「偏差は正, 小さければ負.」のような「だ」省略の名詞述語文（述語がある）

**日本語での正しい書き方**: 述語を付けて閉じる（「〜といった話題だ」）か、直後に受けの文を置く。

**採取実例**:
- 「多重比較の制御, ブートストラップ, 重回帰, 時系列モデルや階層モデル.」→「〜といった話題だ.」（次文が別話題に移り、列を受ける述語がなかった。第10章終節）

**表面形を問わず、この機序に該当するものはすべて禁止。**

## 族未満の採取（機序の括り待ち）

族に括るには実例が足りないが、修正が確定した採取。同種が再発したら族に昇格させる。

- **place/placement の直写（2件・昇格候補）**: 「残差を予測値に対して置く図」→「プロットした図」（第9章）／「どれだけ表示するかという配置」→ 「配置」を削除（第8章）。
- **one を「1つ」と直写（2件・昇格候補）**: 「片方を1つ動かした」→「1単位増えた」／「製作費が1つ増えるごとに」→「1単位増えるごとに」（金額・連続量に助数詞「つ」は使わない。第9章）。
- **同格コンマの直写**: 「必ず点$(\bar{x}, \bar{y})$, データの重心にあたる点を通る」→「データの重心にあたる点$(\bar{x}, \bar{y})$を必ず通る」（"the point, the centroid" 型。日本語は語順を替えるか「すなわち」で受ける。第9章）。
- **読者の身体反応の予告**: 「意外なところで手が止まる」→「線の置き場所が意外に決まらない」（慣用句でも、読者の反応を先回りして断定する語りは読者への直接命令を排する本書の方針と衝突する。主語を対象側へ移す。第9章）。
- **結合の立たない目的語**: 「比較が言い切れる」→「問いにはっきり答えられる」（言い切る対象は言明であって比較ではない。第8章）。
- **語法の無理**: 「$x$が1単位散らばるごとに」→「$x$の散らばり1単位あたり」（散らばりを増分で数えない。第9章）。
- **逆の意味に取れる多義**: 「誰も因果を疑わないが」→「因果を本気にする人はいないが」（「疑う」が「あると疑う」「疑問視する」の両義。第8章）。
- 「その約束の場所がここになる.」→「その約束をここで果たす.」（第10章）。
- 「ここで本書はいったん筆を置く.」→「ここでいったん筆を置く.」（書物主語の擬人化。第10章）。
- 「モデルを当て,」→「モデルを当てはめ,」（縮約による多義。第10章）。

## 走査の手がかり（機械走査用・非網羅）

grep 候補。ヒットは判定前の候補にすぎず、非ヒットは非該当を意味しない。走査時は本リストに加え、リズム原則v4 §3 の未吸収の名指し例も照合対象とする。

**走査から除外してよい範囲**: 図表キャプション（`\caption`）内と脚注内の書誌情報は族4の対象外。数式モード（`$...$`, `\[...\]`, equation 環境）内のカンマは読点として数えない。

- 族1: `止ま`, `止め`, `動く`, `動い`, `動き出`, `走`, `壊れ`, `地図に`（計算・式・議論・結論・変数の共変が絡む文脈のみ。カギ括弧内の装置引用は除外）
- 族2: `先に示した`, `先ほど`, `上で見た`, `上の式`, `前に述べた`, `同じ式`, `同じ値`
- 族3: 一文中の読点3個以上かつ数式接続あり（偽陽性率が高いため優先度低）
- 族4: 句点で終わり述語を含まない地の文
- 族未満: `置く`, `配置`, `1つ`（連続量・金額の文脈のみ）
