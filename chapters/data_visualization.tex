\section{データの可視化 : 数字を「図」で考える}

  \subsection{問い : 数字から“形”を取り戻す}
    表は数字の羅列であり, その要約を与えるのが統計量である.
    とはいえ, データの全容を一望したいのも人情だ.
    それを可能にしてくれるのが\textbf{可視化}である.

    ただし, 一枚の図で「すべて」が分かるわけではない.
    同じ人でも写真の角度で印象が変わるように, 図の作り方（角度・切り取り・尺度・ビン幅）で見え方は変わる.
    ゆえに, 複数の視点と図を行き来して確かめることが基本になる.

    この章では, ランニング例の20商品データ（レビュー数・星評価・購買数）を,
    数字の列から\textbf{形}を取り戻すために図へ写し替え, 何が見え, 何は図だけでは言えないのかを体験する.

  \subsection{目的 : 図で何をできるようにするか}
    可視化のねらいは2つ: \textbf{全体像を素早くつかむ}こと, そして\textbf{仮説を生む}ことだ.
    図にすれば, 「どのあたりが多い」「どれくらい広い」「関係はまっすぐか曲がっているか」「外れ値はどこか」が一目でわかる.
    反対に, 図だけでは\textbf{因果は言えない}し, 「偶然でない」も断定できない（それは後章の推定・検定の役割）.
    本章は, \textbf{最小限の道具}で, 何がわかり何はわからないかを体験する.

  \subsection{理論の最小核 : 目が正確に比べられる形に写す}
    まずは\textit{1秒テスト}. 紙に, 長さが 10 と 12 の線分を2本引いてみる. どちらが長いかは一瞬で分かるはずだ.  
    では, 扇形の角度が $36^\circ$ と $43^\circ$ の2つを描いて同じ問いをしてみる. さっきより迷わないだろうか.  
    \textbf{可視化の本質は, 人の目が最も正確に比べられる属性（位置・長さ）へ, データを写し替えること}にある.

    この「写し替え（マッピング）」は, 変数の性質を保つように選ぶのがコツだ（性質保存の原則）:
    \begin{itemize}
      \item \textbf{量的データ（差や比に意味）} — \underline{位置・長さ}に割り当てる.  
            差を読ませたいなら線分の\textit{長さ}（棒グラフ）, 時系列なら\textit{位置}の連続（折れ線）.  
            棒は長さで量を表すので\underline{ゼロ起点}が原則. ただし変化率や偏差の比較では, 対数軸や差分表示を用い, ゼロ起点にこだわらない場合がある.
      \item \textbf{順序データ（並びに意味, 差は不定）} — \underline{並び（位置の順序）}で見せる.  
            ランキングは1位, 2位, 3位, \ldots と配置し, 棒の長さで「差」を暗示しない（差に意味がないから）.
      \item \textbf{カテゴリデータ（区別のみ）} — \underline{離れた位置}や\underline{色相・形}で識別させる.  
            色は違いを示すだけに留め, 大小の解釈は\textit{高さ（長さ）}や\textit{位置}に担わせる.
    \end{itemize}

    \textit{注意}: 角度や面積での比較は誤差が大きい. 同じ 10\% の差でも, \textbf{扇形（角度）や円の面積}では小さく見えたり大きく見えたりする.  
    細かな差を伝えたい場面では, 位置と長さを優先する（円グラフは「構成の雰囲気」を示す用途にとどめる）.

    最後に, 図は\textbf{一発描きで言い切らない}. 図を作る → 気づく → 切り方や尺度を変えて作り直す, を\textbf{安定性テスト}として回す.  
    例: ヒストグラムはビン幅で形が変わる. 幅を変えても「山の位置」や「歪み」の所見が保たれるかを確かめてから, 結論を口にする.

  \subsection{手法 : 初学者の最小セットと読みどころ}
    はじめに身につけるべきは次の5種類. それぞれ\textbf{何が読めて何が読めないか}を意識して使う.
    \begin{itemize}
      \item \textbf{棒グラフ}（カテゴリ比較）: 高さ＝量. \underline{ゼロ起点}. 並び順を工夫（降順, 論理順）. 例外として, 変化率や偏差の比較ではゼロ起点にこだわらない.
      \item \textbf{折れ線グラフ}（時系列）: 点を順序どおりに結ぶ. 線分は補間で実測ではない点に注意.
      \item \textbf{ヒストグラム}（分布の形）: 山の数, 歪み, 裾, 外れ値の手がかり. ビン幅で見え方が変わる（目安は 10--20 ビン, 余裕があれば Freedman--Diaconis: $h\approx 2\cdot IQR\cdot n^{-1/3}$）.
      \item \textbf{箱ひげ図}（ばらつき/群比較）: 中央・IQR・外れ値で\textbf{頑健}に比較. \underline{各群の件数を明示}.
      \item \textbf{散布図}（2変数の関係）: 方向（正/負）, 形（直線/曲線）, 強さ, 外れ値, クラスター. 第3の変数は色や形で重ねる. 点の重なりが強いときは透明度（alpha）やジッターを用いる.
    \end{itemize}
    ほかにも密度曲線, ヒートマップ, スモールマルチプルなどがあるが, 本章では\textbf{最小5種}に集中する.

  \subsection{実践 : 20商品データを図にしてみる}
    ここでは, 3枚の図で\textbf{発見→問い直し}を体験する. 設定（軸・単位・凡例・ビン幅）を必ず記録する.
    \begin{enumerate}
      \item \textbf{レビュー数のヒストグラム}.  
            どの範囲に集中しているか（山の位置）, 歪み（右裾が長いか）, 外れ値の有無を確認する.  
            \textit{試行}: ビン幅を少し広げたり狭めたりして形の安定性を確かめる.
      \item \textbf{カテゴリー別購買数の箱ひげ図}.  
            家電・書籍・ファッション・食品で, 中央値の違いと散らばり（IQR）の違いを見る. 外れ値が結論を左右していないか注記する.  
            \textit{試行}: 各カテゴリーの件数を凡例に併記し, 件数が少ない群の解釈は保留にする.
      \item \textbf{レビュー数と購買数の散布図（星評価で色分け）}.  
            点の雲の傾き（正の関係か）, まっすぐか曲線か, 例外点（レビュー少・購買多など）を探す.  
            \textit{試行}: 対数軸（購買数）に切り替えて, 低～中程度の差が見やすくなるか確認する.
    \end{enumerate}
    3枚を行き来し, 見えた\textbf{仮説}をメモする（例: 「レビュー数が増えるほど購買が増えるが, カテゴリ差もありそう」など）.

  \subsection{結果と次の一手 : 見えたこと/まだ見えないこと}
    図から言えること（例）:
    \begin{itemize}
      \item レビュー数は右に裾が長い → \textit{平均よりも中央値・IQRが代表に向く}.
      \item カテゴリーで購買数の散らばりが異なる → 回帰で\textit{カテゴリ指標}を入れる根拠.
      \item 散布図は概ね正の傾向だが, 外れ値が関係を左右 → データ起因か事業要因かを\textit{別途確認}.
    \end{itemize}
    しかし, 図だけでは\textbf{偶然か否か}は判断できない. 次章以降で, 標本分布・推定・検定へ橋渡しする.

  \subsection{作業 : 図を作る・記録する・ひと言で要約する}
    以下を実施し, 設定と所見をメモに残す（小数第2位丸め, 単位併記）.
    \begin{enumerate}
      \item レビュー数のヒストグラム（軸範囲, ビン幅, 件数表示の有無）と, 「形」を一文で記述.
      \item カテゴリー別購買数の箱ひげ図（件数凡例, 外れ値表示）と, 群間の違いを一文で記述.
      \item 散布図（色=星評価）と, 関係の向き/形/外れ値の所見を一文で記述. 可能なら対数軸版も作成.
    \end{enumerate}
    \textit{記録の約束}: すべての図で軸ラベル・単位・凡例・切り取り基準（フィルタ/並び替え）を明示する.
