## 章の構成（独自設計）

9章は本書全体の総括章であり, 6ステップ構成は適用しない. 章構成案の「扱う根本概念」（バイアスの類型と手法対応／前提の確認／誤用の代表型／データ倫理）と「データ分析での位置づけ」（分析前・中・後で守る最低限のルール）を出発点に, 以下5節で構成する.

| 節 | 役割 |
|----|------|

## 1. 読者の認知動線（順序付き）

---

### 9.1 章の入口 : 1章の「予告」への答え合わせ

**入口の読者状態**  
8章を読み終えた直後. 回帰の前提崩れの話で「次章で整理する」と渡されている. 2〜8章で手法を積み上げてきたが, 各章末で前提が崩れたらどうなるかの伏線を断片的に受け取ってきた状態. 1章で予告された「正しく計算しても結論が間違いになる例」を覚えているかどうかは半々.

**認知動線**

1. 「ここまでで一通り手法を学んだ. 最後の章は何をするのか」 → 本書全体の総括であり, ここまで積み上げた手法に対応する「外側の地雷」を, ひとつの地図にまとめる章だと提示する.  
   → 回答後の読者状態 : 注意事項リストではなく, 学んできた手法に対応した地図が出ると期待する.

2. 「具体的に何が起きうるのか」 → 1章で先出しした「正しく計算しても結論が間違いになる例」の正体を, 数値とともに明かす（後述の驚きポイント①または②を使う）.  
   → 回答後の読者状態 : 自分はこの間違いに気づけなかった, と認める. ここから先の話を自分事として読む構えになる.

3. 「自分が学んだ手法を疑えと言われたら, どこから疑えばいいのか」 → 疑う観点を3つに整理する. 標本の集め方（サンプリング）, データの選別（生存者）, 解釈の仕方（確証）. 順に9.2で扱う.  
   → 回答後の読者状態 : 残りの章の見取り図を持つ.

**出口の読者状態**  
本章の役割（手法の外側を知る）を理解し, 自分の手で実際に間違える可能性を直視している.

**橋渡し**  
3つの観点のうち, 最も身近で最も誤解されやすい「標本の集め方」から始める, と次節へ送る.

### 9.2 バイアスの3類型 : 標本・生存者・解釈

**入口の読者状態**  
1章の予告例で「正しい計算でも結論は間違いうる」を実感したばかり. しかしまだ, 何種類の落とし穴があるのか, それぞれが本書のどの章と対応するかは見えていない.

**認知動線**

1. 「標本の偏りって, 1章でも触れた話だよね. 何が新しいのか」 → 4章のCLTと5章のCIは「無作為抽出」を前提にしていた. 抽出が偏るとSEも信頼区間も「偏った母集団」の値を測ってしまう. 件数を増やしても解決しない（5章のLiteracy Digest逸話を回収）.  
   → 回答後の読者状態 : サンプリングバイアスは「件数で殴れない」性質だと理解する. 4・5章の前提と接続される.

2. 「では, 取れたデータの中での偏りはどう扱うのか」 → 観測できなかったものを忘れる癖こそが生存者バイアス. 戻ってきた爆撃機の被弾箇所を見て補強する話（驚きポイント②）.  
   → 回答後の読者状態 : 「持っているデータ」の外側にこそ重要な情報があるという視点を得る.

3. 「データを取ったあとは安全か. 解釈ならフェアにできるのでは」 → 確証バイアスは取れたデータからでも歪みを生む. 仮説に合うデータを記憶し, 合わないデータを忘れる. p-ハッキング（後述驚きポイント③）はこれを統計の道具で正当化したもの.  
   → 回答後の読者状態 : 「自分は中立に見ている」という思い込みが最も危険だと理解する.

4. 「3つは別の話なのか, つながっているのか」 → 標本→観測→解釈の流れに沿って起きる. データの「上流／中流／下流」の歪みとして整理する.  
   → 回答後の読者状態 : 3類型が独立した知識ではなく, データの流れの順番に沿った地図だと掴む.

**出口の読者状態**  
3類型をデータの流れの上で位置づけ, 各類型が本書のどの章の前提を崩すかを言える.

**橋渡し**  
バイアスは「データの集め方・選び方・解釈の仕方」の話だった. では, 集めたあと手法を当てるとき, 手法そのものが暗黙に置いている前提が崩れる場面はどう見抜くか, と次節へ送る.

### 9.3 前提が崩れるとき : 独立性・線形性・等分散性

**入口の読者状態**  
3類型でデータの流れ全体を点検する視点を得た. しかし, 「独立性が崩れる」「線形でない」と各章で予告されてきた話の具体的な意味は, まだ手元で組み立て直していない.

**認知動線**

1. 「独立性って何だっけ. 4章のCLTで聞いたが, ピンと来ない」 → 時系列データの例（10ヶ月の月次売上）で見せる. 8章の回帰は各点が独立だと仮定しているが, 月次データでは前月の影響が今月に残る. 残差が独立に散らばらない.  
   → 回答後の読者状態 : 独立性は「並んだデータが互いに無関係であること」と具体的に理解.

2. 「では, 独立でないとどうなるのか」 → SEや決定係数を計算する式は形式的には動くが, 出てくる数字は「不当に小さいSE」「過大なR²」になる. 信頼区間が狭く出すぎる.  
   → 回答後の読者状態 : 数字が出ているから安全, ではないと理解.

3. 「線形性は7・8章でやった. 何が新しいのか」 → 残差プロット（8章末で導入済み）を診断ツールとして再定位する. アンスコムの第2組（放物線）を再掲し, 直線当てはめが意味をなさないケースを思い出させる.  
   → 回答後の読者状態 : 「数字が出る」と「直線を当てるのが妥当」は別物だと整理する.

4. 「等分散性って何だっけ. 6章のWelchで聞いたが」 → Welch法を採ったので等分散の仮定は外したが, 回帰では予測値の大きさによって残差の散らばりが変わる場合がある（扇形パターン）. これは予測の信頼幅の解釈をゆがめる.  
   → 回答後の読者状態 : 「平均的な予測は当たっても, ばらつきが場所によって違えば, 予測区間の意味が変わる」と理解する.

**出口の読者状態**  
3つの前提を診断する目を持ち, 数値が出ても診断なしには信用しない構えができている.

**橋渡し**  
ここまでは「数値が出ているのに前提が崩れている」場面だった. 次は, 計算自体は正しいのに「読み方」が間違っているケースに移る, と送る.

### 9.4 誤用の代表型 : p値の誤読・相関と因果・過度な解釈

**入口の読者状態**  
データの集め方と手法の前提までは点検する視点を得た. しかし「正しく計算したのに結論が間違っていた」典型例の中身は, まだ言語化できていない.

**認知動線**

1. 「p値はちゃんと意味を覚えているはずだが」 → 6章で導入したp値の定義を1文で再確認したうえで, よくある誤読を3つだけ示す（「帰無仮説が正しい確率」「効果が大きい証拠」「p>0.05は効果がない証拠」）. 大標本では実用上無視できる差でもp<0.05が出る具体例を数字で示す（例 : 二群差0.05点, n=10000）.  
   → 回答後の読者状態 : p値は「珍しさ」の指標であり, 「効果の大きさ」「真偽の確率」とは別物だと言い分ける.

2. 「では効果の大きさはどう報告するのか」 → 5章の信頼区間と6章Part Bの効果量を再呼び出し. p値とCIと効果量はそれぞれ別の問いに答えており, セットで報告するのが筋だと整理.  
   → 回答後の読者状態 : 「結果の報告は3点セット（p値・CI・効果量）」を運用ルールとして持つ.

3. 「7・8章でも『相関は因果ではない』と言っていた. ここで何を足すのか」 → 因果と相関を取り違えるパターンを「交絡」「逆因果」「偶然の一致」の3つに分けて見せる. 例として, アイスと水難事故（気温の交絡）, ストーク仮説（疑似相関）.  
   → 回答後の読者状態 : 相関を見たときに「交絡因子は何か, 因果方向はどちらか」と自動的に問う癖がつく.

4. 「過度な解釈ってどんな例があるのか」 → 8章の回帰を例に, 外挿の誤り（広告費200万円）と切片の解釈（広告費0万円で売上119万円）を再回収. 「データの範囲を超えた予測は計算式が動くだけで根拠がない」.  
   → 回答後の読者状態 : 「数字が出る」と「現実に当てはめてよい」は別だ, という距離感を持つ.

**出口の読者状態**  
正しく計算しても結論が間違う典型を3つ手元に持ち, 自分の分析を出すときに「報告の3点セット」「相関の3つの落とし穴」「外挿の禁忌」を点検できる.

**橋渡し**  
ここまでは技術と解釈の話だった. 最後に, データを扱うこと自体の「人と社会への責任」に視野を広げて締めくくる, と送る.

### 9.5 データ倫理と「これから」

**入口の読者状態**  
9.4までで, 技術上の落とし穴は手元の地図に書き込まれた. しかし, データを扱うこと自体に伴う社会的責任の話はまだ. 「ここまで本書を読んできた自分は何ができるようになったのか」を確認したい気持ちもある.

**認知動線**

1. 「倫理というと堅苦しい. これは技術書ではないのか」 → 倫理は「データを扱う人への重荷」ではなく, 9.2〜9.4で見た間違いを社会的に増幅させない仕組みだと位置づけ直す. 毎月勤労統計問題を1段落で紹介し, 「設計の逸脱が社会保障の配分にまで響いた」事例として接続.  
   → 回答後の読者状態 : 倫理は技術の続きであり, 切り離せない話だと理解する.

2. 「具体的に何を守ればいいのか」 → 3点に絞る. 収集時の同意と目的明示, 個人情報の保護, 報告時の透明性（手順・前処理・除外の公開）. 列挙ではなく, 9.2〜9.4で扱った間違いと対応づけて見せる（例 : 透明性が欠ければ確証バイアスを外部から検証できない）.  
   → 回答後の読者状態 : 倫理項目を「9.2〜9.4で学んだ間違いを抑える社会的装置」として再解釈する.

3. 「で, 結局自分は何ができるようになったのか」 → 本書全体の到達点を1段落で振り返る. 要約・形・推論の幅・差の吟味・関係・モデル, そして外側の地雷. 「数字が出る」から「数字を疑える」までの距離を獲得した状態だと言葉にする.  
   → 回答後の読者状態 : 自分が手にしたものを言える. 自信と謙虚さの両方を持つ.

4. 「次は何をすればいいのか」 → 補足概念に挙げた発展（多重比較の制御, VIF, 事前登録, データガバナンス）への入口を1〜2行で示し, 自分の手元のデータで本書の手順を最初から最後まで一度回してみる, という具体的な次の一手を提示する.  
   → 回答後の読者状態 : 教科書の終わりが学びの終わりではなく, 自分の問いに戻る出発点だと感じる.

**出口の読者状態**  
本書の総括として, 手法・前提・解釈・倫理の4層を俯瞰でき, 自分の問いに戻る具体的な次の一手を持つ.

**橋渡し**  
本書の終端. 次の章はないが, 章末で「あなた自身の問いに戻ってほしい」と読者を送り出す.

## 2. 概念の必然性チェーン

9章は新しい数学概念を導入する章ではなく, 2〜8章で築いた前提を体系的に裏返す章である. したがってチェーンは「読者の手元にある手法→その手法が暗黙に前提していたもの→前提が崩れる場面の名前→対処法（または認識）」の順で組む.

---

**チェーン1 : 標本→サンプリングバイアス**

- 直前の読者状態 : 4章でCLTを介してSEを手にし, 5章で信頼区間を計算できる. 標本平均±約2SEで母平均を語れると思っている.
- 壁 : 母集団から無作為に抽出されていなければ, 信頼区間は「偏った母集団」の値を狭く正確に測ってしまう. 件数を増やしても解決しない.
- 必要な概念 : サンプリングバイアス（標本の代表性が崩れる類型）.
- 獲得後の読者状態 : 「nを増やせば精度が上がる」は無作為抽出を前提にした話だと言い分けられる. CIの式が動くかどうかと, CIの結果が信頼できるかは別問題だと理解する.

**チェーン2 : サンプリングバイアス→生存者バイアス**

- 直前の読者状態 : 標本の代表性を疑う視点を持った.
- 壁 : 代表性は「取れたデータの中身」だけを見ても判断できない. 取れなかったデータ（脱落, 不参加, 倒産企業）の存在を見落とす.
- 必要な概念 : 生存者バイアス（観測できないものの欠落）.
- 獲得後の読者状態 : 「手元にあるデータの完全性」と「世界の完全性」を区別する.

**チェーン3 : 生存者バイアス→確証バイアス**

- 直前の読者状態 : データの上流の歪みを2種類見分けられる.
- 壁 : 上流が完璧でも, 下流の解釈が偏れば結論は歪む. 仮説に合うデータを重く見る癖と, p-ハッキングのような統計の道具で正当化された確証バイアスがある.
- 必要な概念 : 確証バイアス（解釈段階の歪み）.
- 獲得後の読者状態 : データの流れの上流・中流・下流のすべてを疑う視点を獲得.

**チェーン4 : バイアスの3類型→前提の診断（独立性）**

- 直前の読者状態 : データの流れ全体を疑える.
- 壁 : では具体的にどの手法のどの仮定が崩れたら何が起きるのか, 個別の対応関係がまだ言語化されていない.
- 必要な概念 : 独立性（4章CLT, 8章回帰の暗黙仮定）と, それが崩れたときのSE・R²の挙動.
- 獲得後の読者状態 : 時系列・クラスタ・繰り返し測定で独立性が崩れる場面を特定でき, SEが過小に出るリスクを意識する.

**チェーン5 : 独立性→線形性→等分散性**

- 直前の読者状態 : 独立性の崩れを診断できる.
- 壁 : 独立であっても, 関係の形（線形性）と残差の散らばり（等分散性）が崩れれば, 8章の回帰は「動くが意味をなさない」.
- 必要な概念 : 線形性（残差プロットでU字を見抜く）, 等分散性（残差プロットで扇形を見抜く）.
- 獲得後の読者状態 : 8章で導入した残差プロットを「診断道具」として運用できる.

**チェーン6 : 前提診断→誤用の代表型（p値の誤読）**

- 直前の読者状態 : 計算が正当に動く前提を点検できる.
- 壁 : 計算が正しく前提も満たされても, 結果の「読み方」が間違えば結論は誤る. p値は最も誤読されやすい指標.
- 必要な概念 : p値の正しい読み方の再確認と, 大標本下の有意性インフレ.
- 獲得後の読者状態 : p値・CI・効果量の3点セットで報告するルールを持つ.

**チェーン7 : 誤用→相関と因果**

- 直前の読者状態 : 数値報告の作法を持つ.
- 壁 : 7・8章で「相関は因果ではない」と聞いた. しかし, では何が相関を生むのかを言えない.
- 必要な概念 : 交絡・逆因果・偶然の一致.
- 獲得後の読者状態 : 相関を見たら自動的に「交絡因子は」「因果方向は」と問う.

**チェーン8 : 相関→過度な解釈→倫理**

- 直前の読者状態 : 解釈の歪みを類型化できる.
- 壁 : 個人レベルの誤読は技術で抑えられるが, 社会レベルでデータが扱われるときには制度的な歯止めが要る.
- 必要な概念 : データ倫理の3点（同意・保護・透明性）.
- 獲得後の読者状態 : 倫理を「9.2〜9.4の歪みを社会的に抑える装置」として運用できる.

### 驚きポイント① : シンプソンのパラドックス（バークレー大学院入試, 1973年）

**配置** : 9.1の認知動線2項目（「具体的に何が起きうるのか」）

**内容** :  
1973年, UC Berkeley大学院は性別差別で訴えられた. 全体の合格率は男子44%, 女子35%. 9ポイント差は大きい. ところが学部別に見ると, 6つの主要学部のうち4学部で女子の合格率が男子と同じか高かった. 全体集計でだけ男子有利に見えていたのだ.

種明かしは応募分布. 女子は人文系（合格率20〜30%）に多く応募し, 男子は理工系（合格率60〜70%）に多く応募していた. 各学部内では公平でも, 女子が「合格率の低い学部」に集中していたため, 全体では男子有利に見えた.

数値で見ると裏切りが効く. 各学部では公平か女子有利, しかし合計したら男子有利. 集計の単位を変えるだけで結論が逆転する.

**読者の直感を裏切る点** : 「全体の数字こそ真実」「層別はノイズ」という素朴な感覚が, 実は逆だという発見.

---

### 驚きポイント② : 第二次大戦の爆撃機とAbraham Wald

**配置** : 9.2の認知動線2項目（生存者バイアスの導入）

**内容** :  
第二次大戦中, 米軍は帰還した爆撃機の被弾箇所を分析していた. 翼の先端と機体後部に弾痕が集中していた. 軍上層部は「弾痕の多い場所を補強せよ」と指示しようとした.

統計学者Abraham Waldはこれを止めた. 「補強すべきは弾痕の少ない場所だ」. なぜか. 帰還できた機は, それらの箇所に被弾しても飛べた機だ. エンジンや操縦席の被弾を受けた機は帰ってきていない. 観測されたのは「生存者」だけで, 死者は弾痕分布に現れない.

**読者の直感を裏切る点** : 「データが集まっている場所にこそ重要な情報がある」という常識が, 「データがない場所こそ情報の核心」へ反転する.

---

### 驚きポイント③ : 16枚のフェアコインで「有意な偏り」を見つける

**配置** : 9.2の認知動線3項目（確証バイアスとp-ハッキング）

**内容** :  
完全に公平なコインを20枚用意し, それぞれ100回投げる. 各コインで「表の出現率は50%と異なるか」をp<0.05で検定する. 20枚すべてが本当に公平でも, 偶然に1〜2枚は「有意」と出る確率が高い（$1-(1-0.05)^{20} \approx 0.64$ : どこかで偽陽性が出る確率は64%）.

研究者が「有意になった1枚」だけを論文に書き, 残り19枚を捨てれば, 「特殊なコインを見つけた」という主張が完成する. 計算は正しい. しかし結論は嘘だ.

**読者の直感を裏切る点** : 「p<0.05が出れば本物」という素朴な信頼が, 「何回試したか」を考慮しないと無意味だと知る. 6章で習った検定の道具が, ここでは「正当化に悪用される道具」として再登場する.

## 4. 具体例候補

---

### 例A : Literacy Digest 1936年大統領選挙誤予測（コラム向き／長め）

**内容** : 1936年, Literacy Digest誌は240万人への大規模調査でランドン候補の勝利を予測した. 結果はルーズベルトの圧勝. 一方, ジョージ・ギャラップはわずか5万人の調査で正しく予測した. Digestのリストは電話帳と自動車登録名簿で, 大恐慌下の貧困層（民主党支持）を体系的に外していた. 件数では数十倍負けていたギャラップが正しかったのは, 抽出の代表性で勝っていたから.

**配置** : 9.2 サンプリングバイアスの認知動線1項目の回答として.

**なぜ刺さるか** : 5章で「偏りはnで解決しない」の格言として既出. 読者は逸話を覚えている. 9章でこれを「サンプリングバイアスの典型例」として再分類すると, 5章で受け取った断片が地図上の一点に位置づけられる安堵感が生まれる.

---

### 例B : アイスクリーム消費量と水難事故の正の相関（本文用／短め）

**内容** : 月別データでアイス消費量と水難事故件数に強い正の相関がある. 「アイスを食べると溺れる」のではなく, 第3変数「気温」が両方を独立に押し上げている. 交絡因子の典型.

**配置** : 9.4 認知動線3項目（相関と因果）の回答として.

**なぜ刺さるか** : 7章で「相関と因果は別物」と聞いたが, 「どう別物なのか」の具体像はぼんやりしていた. ここで「交絡因子」という言葉と一対一に結びつく具体例が来ると, 抽象が肉付けされる.

---

### 例C : 毎月勤労統計問題（コラム向き／長め）

**内容** : 日本の公的統計で, 本来全数調査すべき大規模事業所に抽出調査が適用され, 復元処理が怠られた. 結果として平均賃金が過小に算出され, 雇用保険給付などで延べ約2{,}000万人に総額500億円超の過少給付. 「設計の逸脱」が社会保障の配分という実体的結果に直結した事例.

**配置** : 9.5 認知動線1項目（倫理は技術の続き）の回答として.

**なぜ刺さるか** : 「倫理は他人事の話」と感じている読者に, 9.2のサンプリングバイアスが社会保障に直結した実例を見せることで, 「設計と倫理は同じ硬貨の表と裏」だと体で理解させる. 抽象的な倫理項目（同意・保護・透明性）の前にこの具体例を置くと, 後の項目が「なぜ守るのか」の答えになっている状態で読める.

---

### 例D : ストーク（コウノトリ）の数と出生数の正の相関（本文用／短め）

**内容** : ヨーロッパ各国でコウノトリの巣の数と人間の出生数に正の相関がある. もちろんコウノトリは赤ん坊を運んでこない. 国のサイズ（面積・人口）が両方を押し上げる交絡因子. 「強い相関」を見ても因果に飛ばないための定番例.

**配置** : 9.4 認知動線3項目（相関と因果）の回答として例Bと併せる, または例Bの代替として.

**なぜ刺さるか** : 童話的な笑いがあり, 「相関と因果は違う」を覚えやすい記憶として残せる. 退屈になりがちな最終章にユーモアを差し込む位置として適している（要件定義書 §4.2のトーン要件と整合）.

## 設計メモ

- **章構成案で節構成が指定されていない件** : 9章は1章・6章Part Bと並ぶ6ステップ例外章だが, 章構成案v5では1章・6章Part Bと違って独自構成が明示されていない. そこで「扱う根本概念」の4区分（バイアスの類型／前提の確認／誤用の代表型／データ倫理）に, 章構成案の「物語の役割」（1章への答え合わせ・地雷の地図）と「データ分析での位置づけ」（分析前・中・後）を組み合わせ, 5節構成として再構築した.

- **参照用9章との差分** : 旧版は「図表設計の誤解」を冒頭に置く構成だったが, 章構成案v5の「2〜8章の手法と対応させる」方針に揃え, 図表設計の話は本文から外した（必要なら9.5の透明性の項に1段落で吸収可能）. 代わりに, 1章の「予告」を9.1で回収する構造を本書全体の物語の閉じ方として設計した.

- **驚きポイントの位置取り** : 9章は「素朴な方法と限界」を持たないため, 驚きポイントは9.1（1章への答え合わせ）と9.2（3類型の導入時）に配置した. シンプソン・ウォルド・p-ハッキングの3つは, それぞれが本章で扱う3類型（解釈の歪み／生存者／確証）と一対一に対応するように選んだ.

- **「だから何」テストへの配慮** : 9章は技術を導入しない章なので, 各節末で「読者は何ができるようになったか」を明示することが特に重要になる. 各節の「出口の読者状態」をその答えとして書き起こした. 後段の本文生成エージェントは, 各節末に「ここで読者が獲得した点検眼」を1〜2文で言語化することで, 「注意事項の羅列」ではなく「能力の地図」として読める章にできる.

- **トーンの設計** : 9章は内容の性質上, 「読者を脅す」方向に流れやすい. 要件定義書 §4.2の「皮肉や肩の力を抜かせるユーモア」は特にここで効く. 例D（ストーク）, 驚きポイント③のコイン（「捨てられた19枚」のニヤリ要素）など, ユーモアで肩の力を抜かせる仕掛けを各節に1つは置く想定で具体例を選んだ.
